## Capability matrix

| Capability ID | Capability | Current support | Detailed behavior | Constraints / prerequisites | Roadmap status | Sources | Notes |
| --- | --- | --- | --- | --- | --- | --- | --- |
| AI-001 | AI Helper — supported backends | Supported | Three configurable backends: Azure AI (requires API key + deployment name + endpoint URL), OpenAI GPT-4o (requires API key), Anthropic Claude Opus 4.1 (requires API key). Default model selector in Preferences. No bundled LLM — users supply their own API credentials. | API key must be obtained from Azure / OpenAI / Anthropic independently. Costs are incurred per token with the respective provider. | confirmed | S1 | Edition unverified; assumed all editions that include AI Helper. |
| AI-002 | AI Helper — configuration | Supported | Temperature control (default: low for accuracy; increase for creativity). Disable AI Helper globally via "Enable AI Helper" checkbox in Preferences. | Temperature range and accepted values are unverified. | confirmed | S1 | All editions with AI Helper. |
| AI-003 | AI Helper — natural language query generation | Supported | Integration points: Collection Tab, IntelliShell, Aggregation Editor. Natural language question → MongoDB find query, aggregation pipeline stages, or shell script. Multi-source prompt: select multiple collections from the same database; field checkbox to reduce token count. "Review and change prompt data" dialog with field selection and prompt metadata review. | AI output must be reviewed before execution; generated queries may not be optimal. | confirmed | S1 | All editions with AI Helper. |
| AI-004 | AI Helper — result application | Supported | "Change query" / "Change pipeline" / "Change script" button applies generated result directly to the active editor (Collection Tab query bar, Aggregation Editor stage, or IntelliShell). "Copy snippet" button for clipboard. Pop-out window for full-screen AI Helper view. | Applied result replaces current editor content; undo should be verified before applying. | confirmed | S1 | All editions with AI Helper. |
| AI-005 | AI Helper — session history and restore | Supported | Session history with timestamps stored locally. Session Restore persists history across app restarts. Enables reviewing prior AI conversations without re-querying. | History is local to the Studio 3T installation; not shared across machines or team members. | confirmed | S1 | All editions with AI Helper. |
| AI-006 | AI Helper — keyboard shortcut | Supported | Open AI Helper: Ctrl+Shift+B (macOS: ⌘+Shift+B). Send prompt: Ctrl+Enter. | — | confirmed | S1 | All editions with AI Helper. |
| AI-007 | Local MCP Server — availability and configuration | Supported | Requires Studio 3T 2026.9 or later. Enable via Preferences → External Tools → Local MCP Server toggle. Runs HTTP transport at http://127.0.0.1:27117/mcp (port configurable). No authentication required for local loopback access. | Studio 3T must be running for the MCP server to be active. Port 27117 must be available; configurable if occupied. | confirmed | S1 | Studio 3T 2026.9+. Edition unverified. |
| AI-008 | Local MCP Server — available tools (10 tools) | Supported | 1. list_connections — lists configured MongoDB connections. 2. connect — opens a connection for subsequent tool use. 3. list_databases — lists databases on active connection. 4. list_collections — lists collections in a database. 5. query — runs a find query and returns results. 6. explain_query — explains query execution plan. 7. analyze_schema — describes collection field shapes. 8. list_indexes — lists indexes on a collection. 9. get_collection_statistics — returns size, doc count, related stats. 10. assess_collection_health — summarises indexing and shape consistency. | All 10 tools are read-oriented; write operations are not exposed via the MCP server. | confirmed | S1 | Studio 3T 2026.9+. |
| AI-009 | Local MCP Server — compatible clients | Supported | Verified compatible: VS Code + GitHub Copilot extension, Cursor, Claude Code, Claude Desktop, Cline (VS Code extension), Gemini CLI, and any MCP-compatible client supporting HTTP transport. | Clients using stdio MCP transport only (not HTTP) are not directly compatible without an adapter. | confirmed | S1 | Studio 3T 2026.9+. |
| AI-010 | 3T MCP — standalone binary | Supported | Binary: stt-cli. Distributed via GitHub releases (3tio/3t-mcp-releases). macOS/Linux install: curl installer script. Windows install: PowerShell installer script. Transport: stdio (unlike Desktop IDE's HTTP MCP server). Auth: stt-cli login → browser-based OAuth; credentials stored locally; requires 3T account. | Requires a 3T account (free registration). stdio transport; HTTP-only clients need a bridge adapter. | confirmed | S2 | Standalone product; separate install. |
| AI-011 | 3T MCP — capabilities | Supported | Collection browsing (list all databases and collections), Query execution (run find queries; explain query plan), Schema analysis (full field shapes, types, nested structures), PII scanner (finds fields potentially containing PII across collections). Strictly read-only — no writes, deletes, or modifications. Runs entirely locally — no cloud intermediary, no outbound data flow. | Read-only constraint is by design and not configurable. PII scanner identifies candidate fields by name/pattern heuristics; results require human review. | confirmed | S2 | Standalone product. |
| AI-012 | 3T Build AI Agent | Supported | AI Agent feature available within the 3T Build browser IDE. Context-aware answers based on active dataset. Natural language query generation. | 3T Build is a separate product (browser IDE); availability and edition requirements are unverified for this sub-feature. | unknown/unverified | S2 | 3T Build product; edition requirements unverified. |
