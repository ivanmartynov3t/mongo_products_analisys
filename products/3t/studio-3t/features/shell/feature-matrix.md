## Capability matrix

| Capability ID | Capability | Current support | Detailed behavior | Constraints / prerequisites | Roadmap status | Sources | Notes |
| --- | --- | --- | --- | --- | --- | --- | --- |
| SHELL-engine | Shell engine — mongosh integration | Supported | mongosh/mongo executable bundled with Studio 3T; can be overridden per connection or globally in Preferences > Tools > IntelliShell. Custom path to an external mongosh binary is configurable. Format code (Ctrl+Alt+L). Indent/unindent (Tab/Shift+Tab). Line comments (Ctrl+/). Date tags support. | mongosh binary must be accessible at the configured path; custom path must point to a compatible mongosh version. | confirmed | S1 | All editions. |
| SHELL-autocomplete | Auto-complete | Supported | Auto-completion triggered by Ctrl+Space (and automatically in context). Covers: JavaScript functions and keywords, shell types and methods (db., collection., cursor.), MongoDB aggregation operators, collection names from the active connection, field names from the active collection, shell helper commands. | Auto-complete for field names requires the connection to be open and the collection to be accessible. | confirmed | S1 | All editions. |
| SHELL-validation | Syntax validation | Supported | Live error highlighting as code is typed. Errors indicated by: red gutter crosses (left margin), right-ruler markers (right scrollbar). Errors identified before execution, reducing runtime failures. | Validation is syntactic (JavaScript/mongosh grammar); logical MongoDB query errors are only caught at execution time. | confirmed | S1 | All editions. |
| SHELL-run-all | Run all — F5 | Supported | F5 executes the entire script top-to-bottom. In Shell mode, output appears in Raw shell output tab. In Query Assist mode, each standalone find() or aggregation call opens an editable result tab with a line-number label. | In Query Assist mode, only standalone find() and aggregation pipeline calls open editable result tabs; other output is in the Raw tab. | confirmed | S1 | All editions. |
| SHELL-run-cursor | Run to cursor — F6 | Supported | F6 executes the script from the beginning to the cursor's current line. Useful for step-through debugging of large scripts. | — | confirmed | S1 | All editions. |
| SHELL-run-select | Run selected — F9 | Supported | F9 (or Ctrl+Enter on selected text) executes only the currently highlighted/selected text block. Enables isolated execution of a sub-section of a script without modifying the overall script. | Selection must be syntactically valid on its own for successful execution. | confirmed | S1 | All editions. |
| SHELL-modes | Execution modes — Shell vs Query Assist | Supported | Two modes switchable per IntelliShell tab. Shell mode: all output goes to the Raw shell output tab only; no editable result tabs are opened; useful for administrative scripts with non-query output. Query Assist mode (default for find/aggregation): each standalone find() or aggregation pipeline call opens a dedicated, editable result tab labelled with the source line number; supports inline editing, Visual Explain, Query Code tab, and pinned tabs. Destructive operation warnings (update/delete/drop) displayed in Query Assist mode; suppressible per session. | Query Assist mode result tabs apply only to standalone find() and aggregation pipeline calls; other shell output still appears in the Raw tab. | confirmed | S1 | All editions. |
| SHELL-result-tabs | Result tabs (Query Assist mode) | Supported | In Query Assist mode each query in the script that produces a cursor (find, aggregation) opens a separate result tab. Tab label includes the source line number for traceability. Tabs support: inline document editing (Tree/Table/JSON view), pin tab (keep open across re-runs), close individual tabs, re-run from tab. | Only available in Query Assist mode. Result tabs for the same script are replaced on each re-run unless pinned. | confirmed | S1 | All editions. |
| SHELL-result-views | Result view modes | Supported | Each result tab supports the full Studio 3T result view suite: Tree View (hierarchical), Table View (rows/columns with inline edit), JSON/BSON View (raw document), Query Code tab (generated MongoDB Shell, Node.js, Java, C#, Python, PHP, Ruby code), Visual Explain tab (explain plan visualization). | Visual Explain is only available in Query Assist mode; not available in Shell mode. | confirmed | S1 | All editions. |
| SHELL-save-load | Save & load scripts | Supported | Save current script to Query Manager (as IntelliShell Script query type, with optional connection/DB context) or as a .js file. Load script from Query Manager or from a .js file on disk. Filename date/time placeholders usable in script output paths for unique filenames per run (e.g., #today). | .js file saves do not retain Studio 3T connection context; Query Manager saves retain full context. | confirmed | S1, S2 | All editions. |
| SHELL-integrations | Tool integrations | Supported | Inbound: "Open in IntelliShell" available from Collection Tab (sends current query), Aggregation Editor (sends pipeline as mongosh syntax), Query Profiler (opens profiled query). Outbound: Visual Explain tab available inside IntelliShell result tabs; "Open in IntelliShell" available from Aggregation Editor Query Code tab; IntelliShell Script Tasks can be scheduled via the Task Scheduler (Pro/Base+). | "Open in IntelliShell" from Aggregation Editor sends the pipeline as mongosh-format JavaScript. Scheduling requires Pro/Base+ edition. | confirmed | S1, S2 | All editions for integrations; Task Scheduler scheduling requires Pro/Base+. |
