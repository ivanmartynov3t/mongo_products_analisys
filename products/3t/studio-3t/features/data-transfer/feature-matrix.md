## Capability matrix

| Capability ID | Capability | Current support | Detailed behavior | Constraints / prerequisites | Roadmap status | Sources | Notes |
| --- | --- | --- | --- | --- | --- | --- | --- |
| TRANSFER-import-csv | Import — CSV | Supported | Options: delimiter, text qualifier, skip N header lines, header detection (or auto-generated field_0..n names). Empty field handling: Null, empty string, or exclude. Trim spaces, unescape control characters. Flat document option (dots → underscores, disables nested object creation). Array detection disable. Field type override per column. Date formats: ISO, Locale, MDY, DMY, YMD, MYD, DYM, YDM with /, -, . separators. Clipboard import. | CSV files must be UTF-8 or compatible encoding; encoding issues may produce field name/value corruption. | confirmed | S1 | All editions. |
| TRANSFER-import-json | Import — JSON | Supported | Multiple source files or clipboard. JSON validation option (scans file twice; slows import but catches malformed JSON before insertion). Insertion modes (insert, upsert, merge). | JSON validation doubles import time on large files. | confirmed | S1 | All editions. |
| TRANSFER-import-bson | Import — BSON / mongodump | Supported | Select root mongodump folder or archive file. Uses bundled mongorestore executable (overridable via Preferences). Single database/collection import from archive supported. BSON options including compression. | mongorestore binary must be accessible; version mismatch with MongoDB server may cause issues. | confirmed | S1 | All editions. |
| TRANSFER-import-sql | Import — SQL database source | Supported (Pro/Base+) | SQL Connection Manager; source: full table or custom SQL query. NULL handling configuration. Error handling (stop on error or continue). Array creation from numeric dotted names. Embedded document creation from dotted paths. Field type mapping. Data masking rules applicable at import time. Supported databases: MySQL, PostgreSQL, Oracle, SQL Server. | Requires Pro/Base+ edition. JDBC connection to source database required. | confirmed | S1 | Pro/Base+ only. |
| TRANSFER-import-mongo | Import — Another MongoDB collection | Supported | Source and target can be different connections, databases, or collections. Insertion modes (insert, upsert, merge, replace). Data masking applicable. | Requires network access to both source and target MongoDB instances. | confirmed | S1 | All editions. |
| TRANSFER-import-modes | Import — insertion modes | Supported | Insert (fail on duplicate _id), Insert (ignore duplicate _id), Upsert (update or insert by _id), Merge (merge fields), Replace (replace document). | Upsert and Merge require the _id field (or specified match field) to be present in source documents. | confirmed | S1 | All editions. |
| TRANSFER-export-src | Export — sources | Supported | Entire collection, entire view, current find() query result (editable inside Export unit tab), current aggregation query result, current cursor, or specific selected documents. | Aggregation export reflects the current pipeline state at export time. | confirmed | S2 | All editions. |
| TRANSFER-export-csv | Export — CSV | Supported | Partial or full scan (configurable). Custom field list, delimiter, text qualifier, record separator. Presets (save/load export configurations). Null value handling. Date/time placeholders in file path for unique filenames per run. Incremental execution support. | Large collection CSV exports may time out on low-memory servers. | confirmed | S2 | All editions. |
| TRANSFER-export-excel | Export — Excel (.xlsx) | Supported | .xlsx format. Column headers auto-generated from field names. Auto-sizing columns. Text alignment configuration. | Excel row limit (1,048,576 rows) applies; collections exceeding this require filtering or splitting. | confirmed | S2 | All editions. |
| TRANSFER-export-json | Export — JSON | Supported | MongoDB Shell output format or mongoexport-compatible format. Options: commas between documents, document array wrapping. | — | confirmed | S2 | All editions. |
| TRANSFER-export-bson | Export — BSON / mongodump | Supported | Folder or archive output. Compression options. Document recovery mode. | Uses mongodump-compatible format; compatible with mongorestore for restoration. | confirmed | S2 | All editions. |
| TRANSFER-export-sql | Export — SQL | Supported (Pro/Base+) | SQL targets: MySQL, Microsoft SQL Server, Oracle, PostgreSQL. Field-to-column mapping. SQL table name override. CREATE TABLE and DROP TABLE IF EXISTS statement options. SQL type override per field. | Requires Pro/Base+ edition. Target SQL database connection required. | confirmed | S2 | Pro/Base+ only. |
| TRANSFER-export-mongo | Export — Another MongoDB collection | Supported | Cross-server, cross-database collection export/copy. Export modes: insert, upsert, merge, replace. | Requires network access to target MongoDB instance. | confirmed | S2 | All editions. |
| TRANSFER-incremental | Incremental export | Supported | Configure tracking field (must be unique + sortable; default _id). First run: starts from first document (ascending by tracking field); state saved. Subsequent runs: adds {$gt: <saved_state>} to query transparently. Up to 5 historical resume points selectable. Reset to fresh state option. Works with file and database targets. Does NOT track updates to existing documents. | Tracking field must be unique and sortable. Updates to existing documents are not captured by incremental logic. | confirmed | S2 | All editions. |
| TRANSFER-masking-tool | Data masking — standalone tool | Supported (Pro/Base+) | Field-level obfuscation. Source → target collection (default: <<name>>_masked) or overwrite source (disables _id masking). Multiple masking units per job (drag collections from Connection Tree). JSON preview updates dynamically as rules change. Exception logs (type mismatches, warning thresholds). History tab. Rescan collection (default last 50 docs). Add sibling/child fields manually. Dropdown: show all / masked only / unmasked only. | Requires Pro/Base+ edition. Overwrite mode is irreversible without backup. | confirmed | S3 | Pro/Base+ only. |
| TRANSFER-masking-types | Data masking — techniques by BSON type | Supported (Pro/Base+) | All types: Not masked, Null out field, Exclude field, Shuffle. String: Show only first/last N chars; mask entire string; mask substrings matching regex; scramble characters; replace with fixed string; replace with random string. Numeric (Double, Int32, Int64, Decimal128): Substitute with fixed number; add percentage; subtract percentage. Date: Substitute with random date/time; substitute with fixed date/time. Array: Empty contents. Boolean: Negate; substitute with fixed boolean. ObjectId: Substitute with new ObjectId. | Masking technique availability depends on field BSON type; type mismatches are logged to exception log. | confirmed | S3 | Pro/Base+ only. |
| TRANSFER-masking-inline | Data masking during import/export | Supported (Pro/Base+) | Data masking rules can be applied during Import Wizard (SQL source) and Export Wizard runs. Original source data is not modified — masking is applied in-flight to the output. | Requires Pro/Base+ edition. | confirmed | S3 | Pro/Base+ only. |
| TRANSFER-task-save | Save as task / scheduling | Supported (Pro/Base+) | Import and Export configurations can be saved as tasks and executed on-demand or scheduled via the Task Scheduler. Progress visible in the Operations pane. | Requires Pro/Base+ edition. | confirmed | S2 | Pro/Base+ only. |
